Paybacker UAT & UX Test Plan | v1.0 | April 2026 Introduction This document contains the complete User Acceptance Testing (UAT) and User Experience (UX) test plan for Paybacker (paybacker.co.uk), an AI-powered savings platform for UK consumers. Paybacker helps users dispute unfair bills with AI-generated complaint letters citing UK consumer law, track subscriptions and recurring payments, scan bank accounts and email inboxes for hidden costs, and take control of their finances. Purpose This test plan is designed for external QA testers to systematically test every feature of the Paybacker platform. It covers functional testing (UAT), cross-browser/device testing, security testing, and subjective UX quality assessment. How to Use This Document Work through each section in order. For every test case: read the preconditions and steps carefully, execute the steps exactly as described, record your result (Pass/Fail) in the Pass/Fail column, and if a test fails, create a detailed bug report (see Bug Reporting section below). Test Environment URL: https://paybacker.co.uk Test Account: Will be provided separately via Upwork message Browsers Required: Chrome (latest), Firefox (latest), Safari (latest if available), Mobile Safari, Android Chrome Devices: Desktop (1920x1080), Tablet (768px width), Mobile (375px width) Bug Reporting Format For each failed test, please create a bug report with the following information: 1. Test Case ID (e.g. LP-001) 2. Browser and device used 3. Steps to reproduce (exact steps you took) 4. Expected result vs actual result 5. Screenshot or screen recording 6. Severity: Critical / High / Medium / Low 7. Any console errors (open DevTools > Console tab) Please submit all bug reports in a single document or spreadsheet at the end of testing, grouped by severity. CONFIDENTIAL | Page 3
